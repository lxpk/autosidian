# Support

## Getting help

- **Documentation** — Start with [README.md](README.md) and [SPEC.md](SPEC.md) for install steps and expected behavior. [UI.md](UI.md) lists settings in detail.
- **Issues** — When the project hosts a public issue tracker, use it for bug reports and feature requests. Include Obsidian version, Autosidian version, and the versions of [Folder Notes](https://github.com/LostPaul/obsidian-folder-notes), [Waypoint](https://github.com/IdreesInc/Waypoint), [Iconize](https://florianwoelki.github.io/obsidian-iconize/), and [Pixel Banner](https://github.com/jparkerweb/pixel-banner) if relevant.
- **Autosidia (registry web app)** — For problems specific to the sharing platform (login, upload, or browsing presets), use the support channel that ships with that service (to be linked from the Autosidia site and from [SPEC.md](SPEC.md#autosidia-community-registry) when live).

## What we need in a good bug report

- Steps to reproduce.
- What you expected vs what happened.
- Whether the issue happens with **retroactive** or **new-only** features (rate limits, large vaults).
- A minimal description of vault structure (folder depth, number of files) if performance-related.

## Security

If you find a **security** issue, do **not** open a public issue. Follow [SECURITY.md](SECURITY.md).
